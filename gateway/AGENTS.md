# 柒月·网关 — 工作流

> ⚠️ 网关不调用任何 LLM。所有推理由思维层完成。

## 核心职责

### 1. 多平台消息接入
- 钉钉 (DingTalk): Webhook + 机器人 API
- 微信 (WeChat): 企业微信机器人
- 飞书 (Feishu): WebSocket 长连接
- 统一消息格式，屏蔽平台差异

### 2. 秒回 ACK
- 收到消息立即硬编码回复"收到，正在处理 🔍"
- 不等思维层，~50ms 完成
- 纯规则匹配，无 LLM

### 3. 消息转发
- 原始消息原封不动打包
- 通过 EventBus 推送给思维层（Brain）
- 不做任何预处理/意图猜测/参数提取

### 4. 结果推送
- 收到思维层的回复后
- 通过原通道推送给用户
- 不做二次格式化或修改

### 5. 统一会话持久化
- 所有消息存入会话存储
- 支持跨通道共享上下文
- 保持 120 条消息上限，超出后自动压缩

---

## 完整工作流程

```
用户输入 (钉钉/微信/飞书)
    │
    ▼
[秒回 ACK]  ← 硬编码，~50ms，无 LLM
    ├── "收到，正在处理 🔍"
    │
    ▼
[消息转发]  ← EventBus "message.received"
    ├── 原始消息原封不动
    └── → 思维层 (Brain)
    │
    ▼
        [思维层处理]
        ├── 意图识别 (LLM)
        ├── 任务规划 (LLM)
        └── 回复生成 (LLM)
    │
    ▼
[接收回复]  ← EventBus "response.ready"
    │
    ▼
[结果推送]
    ├── 通过原通道推送给用户
    └── → 会话持久化
```
